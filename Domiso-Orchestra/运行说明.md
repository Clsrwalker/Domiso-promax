# Domiso Orchestra 运行说明

这个文件夹是联网合奏版的独立实现。控制电脑只负责上传谱面、分配轨道和发送统一开始时间；每台演奏电脑本地执行自己分到的轨道。

## 1. 放到 Downloads

把整个 `Domiso-Orchestra` 文件夹放到每台电脑的 Downloads 目录：

```powershell
cd "$env:USERPROFILE\Downloads\Domiso-Orchestra"
python -m pip install -r requirements.txt
```

## 2. 控制端电脑

在控制端电脑运行：

```powershell
cd "$env:USERPROFILE\Downloads\Domiso-Orchestra"
python -m domiso_orchestra.conductor --host 0.0.0.0 --port 8765
```

控制端自己打开：

```text
http://127.0.0.1:8765
```

其他电脑打开：

```text
http://控制端IP:8765
```

控制端 IP 可以用这个命令看：

```powershell
ipconfig
```

## 3. 演奏电脑 A / B

先用 dry-run 测试连接，不会真正按键：

```powershell
cd "$env:USERPROFILE\Downloads\Domiso-Orchestra"
python -m domiso_orchestra.player_client --server ws://控制端IP:8765/ws/player --client-id PC-A --backend dry-run
```

第二台电脑：

```powershell
cd "$env:USERPROFILE\Downloads\Domiso-Orchestra"
python -m domiso_orchestra.player_client --server ws://控制端IP:8765/ws/player --client-id PC-B --backend dry-run
```

确认控制台网页里能看到 `PC-A` 和 `PC-B` 后，再换成真实按键：

```powershell
.\run_player_admin.ps1 -Server ws://控制端IP:8765/ws/player -ClientId PC-A -Backend ahk-tap -WindowExe Sky.exe
```

第二台电脑：

```powershell
.\run_player_admin.ps1 -Server ws://控制端IP:8765/ws/player -ClientId PC-B -Backend ahk-tap -WindowExe Sky.exe
```

真实按键后端必须管理员权限。`run_player_admin.ps1` 会自动弹 UAC 提权，让 Python 和 AutoHotkey 都以管理员权限运行；如果直接用普通 PowerShell 启动真实按键后端，客户端会直接报错退出。

`--window-exe Sky.exe` 会按进程名自动切到光遇窗口，避免中文窗口标题编码问题。如果找不到窗口，就先手动点到游戏窗口再开始。

`--backend ahk-tap` 是 Sky 默认推荐后端，每个音都会作为短按发送。`--backend ahk` 会使用仓库里的 `Domiso/ahk_compiler/AutoHotkeyU64.exe`，发送更接近原 Domiso 的 down/up 按键。如果某个游戏只接受 Python 后端，可以备用试：

```powershell
python -m domiso_orchestra.player_client --server ws://控制端IP:8765/ws/player --client-id PC-A --backend windows-input --window-exe Sky.exe
```

如果已经手动用管理员 PowerShell，可以不用脚本，直接运行：

```powershell
python -m domiso_orchestra.player_client --server ws://控制端IP:8765/ws/player --client-id PC-A --backend ahk-tap --window-exe Sky.exe
```

## 4. 查窗口和本地测试

查看可匹配的窗口标题：

```powershell
python -m domiso_orchestra.player_client --list-windows --window-exe Sky.exe
```

不连控制端，直接给当前机器发 3 次测试按键：

```powershell
.\run_local_pulse_admin.ps1 -Backend ahk-tap -WindowExe Sky.exe -Key y
```

如果已经手动用管理员 PowerShell，可以直接运行：

```powershell
python -m domiso_orchestra.player_client --local-pulse --backend ahk-tap --window-exe Sky.exe --local-pulse-key y
```

安全 dry-run 版本：

```powershell
python -m domiso_orchestra.player_client --local-pulse --backend dry-run --local-pulse-key y
```

## 5. 控制台流程

1. 上传 JSON 项目，或者直接上传多个 `.txt` / `.dms` / `.mid` 文件生成项目。
2. 在 Tracks 表里把不同轨道分给 `PC-A`、`PC-B`。
3. 点 `Prepare / Load Tracks`。
4. 等客户端显示 `READY`。
5. 点 `Start +5s`。

如果要从歌曲中间开始，先设置 `Start pos ms` 或拖动旁边的滑条，再点 `Start +5s`。`Use Progress` 会把当前最靠后的客户端进度填到起始位置，适合排练长曲时从暂停/停止附近重来。

控制台网页获得焦点时，可以用这些快捷键：

- `F7` 暂停
- `F8` 停止
- `F9` 从头倒计时开始
- `F10` 继续

每台电脑的延迟可以在控制台里调 `Delay ms`：

- `-20` 表示这台电脑提前 20 ms。
- `+20` 表示这台电脑延后 20 ms。
